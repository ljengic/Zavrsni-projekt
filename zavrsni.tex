\begin{sazetak}
U ovom radu opisana je modifikacija operacijskog sustava za rad u stvarnom vremenu FreeRTOS koja omogućuje podršku za model strogih zadataka s ublaženim uvjetima (engl. \textit{weakly hard real-time system}).
U FreeRTOS je ugrađena podrška za inicijalizaciju i kontrolu periodičnih zadataka. Korišten je model strogog sustava za rad u stvarnom vremenu 
s ublaženim uvjetima te strategija prekidanja poslova.
Nadalje, implementirani su različiti algoritmi za raspoređivanje zadataka. Algoritmi su ispitivani i uspoređivani kroz simulacije na velikom 
broju skupova zadataka. Testni skupovi zadataka generirani su posebnom skriptom za generiranje velikog broja skupova zadataka sa zadanim parametrima, koja je implementirana u sklopu ovog rada. 
Dobiveni rezultati su u skladu s očekivanjima te se poklapaju s iznesenim teorijskim razmatranjima i postojećim rezultatima koji su poznati iz literature.

\kljucnerijeci{sustavi za rad u stvarnom vremenu, FreeRTOS, raspoređivanje zadataka, \textit{skip-over model}, strategija prekidanja poslova.}
\end{sazetak}

\engtitle{Modification of FreeRTOS for Control Applications }
\begin{abstract}
This thesis describes a modification of FreeRTOS which implements the weakly hard real-time system model. 
Firstly, software support for initializing and controlling the execution of periodic real-time tasks is implemented.
The execution of periodic real-time tasks is implemented according to the weakly hard real-time model and job killing strategy.
Further, various scheduling algorithms are implemented. The performance of the algorithms is evaluated and compared by simulating the execution of a large number of synthetically generated task sets.
The task sets for testing are generated through a script for generating a large number of task sets with predefined parameters, which is also implemented in this work.
The obtained results coincide with the described theoretical remarks and previous results known from the literature.


\keywords{Real time systems, FreeRTOS, task scheduling, skip-over model, job killing.}
\end{abstract}
